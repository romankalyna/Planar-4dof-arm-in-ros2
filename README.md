# Planar 4-DOF Robot Arm (ROS2, Python)

Simulation and PID control of a planar 4-DOF serial robot arm in Python with ROS2 Humble and Python 3.10.  
Includes full dynamics, gravity compensation, forward/inverse kinematics, data logging, and RViz visualization via URDF.

> This repository contains a complete ROS2 Python project for simulating, controlling, and analyzing a planar 4-DOF (R-R-R-R) robotic arm operating in the XZ plane.


## Features

- **Full robot dynamics simulation with gravity, damping, and optional Coriolis**
- **Joint-space PD/PID control with gravity compensation**
- **Data logging and visualization tools (Matplotlib, RViz)**
- **Inverse and forward kinematics (IK/FK) in Python**
- **All code and nodes run in ROS2 Humble with Python 3.10**


## Tested On

- **ROS2 Humble**
- **Python 3.10**

## Installation 

1. **Clone this repository and enter the workspace folder:**
    ```bash
    git clone https://github.com/romankalyna/Planar-4dof-arm-in-ros2.git
    cd Planar-4dof-arm-in-ros2  # or 'ros2_ws' if you rename it
    ```

2. **Install Python dependencies:**  
    This project requires `numpy` and `matplotlib`.  
    If you don't have them, install with:
    ```bash
    pip install numpy matplotlib
    ```

3. **Install ROS dependencies:**  
    (recommended if building on a fresh ROS2 install)
    ```bash
    rosdep install --from-path src --ignore-src -y
    ```

4. **Build the workspace using colcon:**
    ```bash
    colcon build
    ```

5. **Source the workspace** (do this in every new terminal before running ROS 2 nodes):
    ```bash
    source install/setup.bash
    ```

---

## How to Run

Make sure you are in your workspace‚Äôs root and have sourced `install/setup.bash` as above (**in each terminal**).

1. **Launch RViz for visualization**:
    ```bash
    ros2 launch src/arm_control/launch/display.launch.py
    ```

2. **Run the simulation node** (new terminal, source first):
    ```bash
    ros2 run arm_sim arm_sim_node
    ```

3. **Run the control node** (new terminal, source first):
    ```bash
    ros2 run arm_control arm_control_node
    ```

4. **Run the inverse kinematics (IK) node** (new terminal, source first):
    ```bash
    ros2 run arm_control ik_node
    ```

5. **Send a target end-effector pose** (in any terminal, source first):
    ```bash
    ros2 topic pub /target_ee_pose std_msgs/msg/Float64MultiArray "{data: [0.6, -0.2, 0.1]}"
    ```
